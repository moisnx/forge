CLI Commands

Forge provides a simple command-line interface for managing your static site projects.

forge init

Initialize a new Forge project

REQUIRED 
forge init [project-name] [options]
project-name Name of your new project directory
--template Use a specific project template

forge serve

Start the development server with hot-reload

DEVELOPMENT 
forge serve [options]
--port Specify server port (default: 8080)
--host Specify host address (default: localhost)
--no-reload Disable hot-reload functionality

forge build

Build your site for production

PRODUCTION 
forge build [options]
--output Specify output directory (default: build/)
--clean Clean output directory before building

forge clean

Remove build artifacts and cache 

forge clean

Template Syntax

Forge uses a Jinja2-inspired template syntax that's powerful yet easy to learn.

Variables

Access data from frontmatter, site config, or collections using double curly braces. 

<h1>{{ page.title }}</h1>

<p>By {{ page.author }} on {{ page.date }}</p>
<p>Site: {{ site.title }}</p>

Conditionals

Use if/else statements to conditionally render content. 

{% if page.featured %}

<span class="badge">Featured</span>
{% elif page.draft %}
<span class="badge">Draft</span>
{% else %}
<span class="badge">Published</span>
{% endif %}

Loops

Iterate over collections, lists, or arrays. 

<ul>

{% for post in collections.posts %}
<li>
<a href="{{ post.url }}">{{ post.title }}</a>
</li>
{% endfor %}
</ul>

Template Inheritance

Extend base templates and override specific blocks.

Base Template (base.html): 

<html>

<head>
{% block head %}
<title>Default Title</title>
{% endblock %}
</head>
<body>
{% block content %}{% endblock %}
</body>
</html>

Child Template (page.html): 

{% extends "base.html" %}

{% block head %}
<title>{{ page.title }}</title>
{% endblock %}

{% block content %}
<h1>{{ page.title }}</h1>
{{ content }}
{% endblock %}

Frontmatter

Add metadata to your content files using YAML frontmatter at the top of any markdown file. 

---

title: My Blog Post
date: 2025-10-29
author: John Doe
tags: [tutorial, forge, static-site]
featured: true
description: Learn how to use Forge
---

# Your markdown content here...

Common Frontmatter Fields

title Page title (used in templates and metadata)
date Publication date (ISO format: YYYY-MM-DD)
author Content author name
tags Array of tags for categorization
draft Boolean - exclude from production builds
template Override default template

Configuration

Configure your Forge site using a forge.yaml file in your project root. 

site:

title: My Awesome Site
description: A site built with Forge
author: John Doe
url: https://example.com

paths:
content: content/
templates: templates/
static: static/
output: build/

server:
port: 8080
host: localhost

build:
exclude_drafts: true
minify: true

All configuration values are accessible in templates via {{ site.* }}
